# Product Scope & UX Notes

## 1. Goal & Positioning
- Запустить веб-first nonogram игру (PWA) с функциональностью уровня Easybrain Nonogram / Picture Cross.
- Поддержать офлайн, синхронизацию прогресса, live-ops (daily/seasonal), монетизацию через рекламу и премиум подписку.
- Мобильные клиенты (iOS/Android) планируются позднее, но UX/контент должен быть готов к портированию.

## 2. Reference Insights
| Источник | Ключевые наблюдения |
| --- | --- |
| https://nonogram.com / Easybrain apps | Цветные и ч/б пазлы, ежедневки, коллекции, сезонные карточки, реклама + подписка, мощные подсказки.
| Picture Cross (App Store / Google Play) | Большие пакеты с прогрессом по карте, кат-сцены, энергосистема, рекламные награды.
| Goobix / Puzzle-Nonograms (web) | Лёгкие веб-версии 5×5–15×15; простой UX, без аккаунтов и монетизации. Хороши как ориентир для минимальной сетки.

## 3. Предлагаемый функциональный объём (web MVP)
### Gameplay
- Только ч/б nonograms на старте; сетки 5×5–15×15 (как у веб-референсов).  
- Drag/paint управление, кроссики, отметка ошибок, подсказки уровня конкурентов (hint, auto-line, cross cleanup, mistake highlight).  
- Доступные сложности: Easy / Normal / Hard / Expert; фильтры по размеру.

### Контент и live-ops
- Тематические паки (20–50 пазлов) аналогично Nonogram.com/Picture Cross.  
- Daily Challenge (1–3 пазла в день) с наградами.  
- Генерация случайного пазла по запросу (если технически возможно) поверх паков.  
- Магазин паков + “Featured” раздел; сезонные события по roadmap.

### Прогресс, профиль, UGC
- Сохранение состояния каждой сетки, аккаунт/гостевой режим, синхронизация.  
- Общедоступный редактор пазлов (доступен всем пользователям): 
  - пользователь создаёт пазл, сохраняет на сервере;  
  - генерируется уникальный URL, которым можно поделиться;  
  - пазл доступен всем, но играется в отдельной “community” зоне.  
- Достижения/медали.

### Монетизация
- Free tier: interstitial + rewarded ads (подсказки, продолжение) как у конкурентов.  
- Premium подписка: убирает рекламу, даёт x2 награды, открывает эксклюзивные пакеты (цену/сроки возьмём по market benchmarks).  
- Разовые продажи паков/валюты по аналогии с Nonogram.com.

### Реклама & anti-adblock
- IMA/AdSense интеграция, interstitial после пазлов, rewarded для подсказок.  
- При обнаружении adblock — баннер + ограничение части функций до отключения или покупки Premium.

### Технические обязательства (для дизайна)
- PWA-совместимость (офлайн, иконки, responsive).  
- IndexedDB кеш пазлов и пользовательских уровней.  
- Тёмная тема и варианты для пользователей с нарушениями цветового зрения (в будущем для цветных пазлов).

## 4. UX / Design Deliverables
1. **User flows**: onboarding, выбор пакета, игровой экран, daily challenge, магазин, paywall, настройки.  
2. **Компоненты**: сетка (canvas/DOM), панель подсказок, панель цветов, прогресс по паку, карточки ивента.  
3. **Брендинг**: логотип, цветовая палитра, типографика, иллюстрации для паков/ивентов.  
4. **Реклама UX**: баннеры, interstitial overlay, rewarded модалка, adblock notice.  
5. **Premium UX**: призыв купить, экран подписки, отличительные бейджи.

## 5. Открытые вопросы
- Конкретные цифры по контенту (точное количество паков/пазлов на MVP, расписание добавления цветных паков).  
- Точная экономика подсказок и цены подписки/паков (ориентируемся на конкурентов, но нужно финальное решение).  
- Приоритет языков на релиз (конкуренты идут глобально, нужно подтвердить список).  
- Требования по accessibility (поддержка color-blind режимов, крупные элементы) — особенно актуально перед запуском цветных пазлов.  
- Дополнительные ограничения/moderation для пользовательских пазлов (контент-фильтры, жалобы).

> Остальное (подсказки, реклама, подписка, локализация, редактор) — “как у конкурентов” и зафиксировано в соответствующих разделах.
