# Product Scope & UX Notes

## 1. Goal & Positioning
- Запустить веб-first nonogram игру (PWA) с функциональностью уровня Easybrain Nonogram / Picture Cross.
- Поддержать офлайн, синхронизацию прогресса, live-ops (daily/seasonal), монетизацию через рекламу и премиум подписку.
- Мобильные клиенты (iOS/Android) планируются позднее, но UX/контент должен быть готов к портированию.

## 2. Reference Insights
| Источник | Ключевые наблюдения |
| --- | --- |
| https://nonogram.com / Easybrain apps | Цветные и ч/б пазлы, ежедневки, коллекции, сезонные карточки, реклама + подписка, мощные подсказки.
| Picture Cross (App Store / Google Play) | Большие пакеты с прогрессом по карте, кат-сцены, энергосистема, рекламные награды.
| Goobix / Puzzle-Nonograms (web) | Лёгкие веб-версии 5×5–15×15; простой UX, без аккаунтов и монетизации. Хороши как ориентир для минимальной сетки.

## 3. Предлагаемый функциональный объём (web MVP)
### Gameplay
- Ч/б и (опционально на запуск) цветные nonograms; сетки от 5×5 до 25×25.
- Drag/paint управление, кроссики, отметка ошибок, подсказочные режимы (line solve, single cell, auto-cross).
- Сложности: Easy / Normal / Hard / Expert; фильтры по размеру/цвету.

### Контент и live-ops
- Пакеты (theme packs) по 20–50 пазлов, коллекционные изображения.
- Daily Challenge (1–3 пазла в день) с наградами.
- Сезонные события (2–4 недели) → эксклюзивные открытки/иконки.
- Магазин паков + “Featured” раздел.

### Прогресс и профиль
- Сохранение состояния каждой сетки, возможность продолжить.
- Аккаунт/гостевой режим, синхронизация через backend.
- Достижения/медали.

### Монетизация
- Free tier: interstitial + rewarded ads (подсказки, продолжение).  
- Premium: подписка “без рекламы + x2 награды + эксклюзивные пакеты” и/или разовые покупки паков.  
- Встроенный стор для паков/подсказочных наборов.

### Реклама & anti-adblock
- Веб-интеграция IMA/AdSense, detection ad-block → мягкий paywall или предложение премиума.

### Технические обязательства (для дизайна)
- PWA-совместимость (офлайн, иконки, responsive).  
- IndexedDB кеш пазлов → UX должен учитывать “синхронизацию”.  
- Тёмная тема (желательно сразу) для night-mode.

## 4. UX / Design Deliverables
1. **User flows**: onboarding, выбор пакета, игровой экран, daily challenge, магазин, paywall, настройки.  
2. **Компоненты**: сетка (canvas/DOM), панель подсказок, панель цветов, прогресс по паку, карточки ивента.  
3. **Брендинг**: логотип, цветовая палитра, типографика, иллюстрации для паков/ивентов.  
4. **Реклама UX**: баннеры, interstitial overlay, rewarded модалка, adblock notice.  
5. **Premium UX**: призыв купить, экран подписки, отличительные бейджи.

## 5. Открытые вопросы
1. Объём контента на запуск (сколько паков/пазлов, нужны ли цветные сразу?).  
2. Конкретные подсказки и их стоимость (экономика).  
3. Модель подписки/цены, нужны ли разовые покупки паков.  
4. Сценарии рекламы (частота interstitial, условия rewarded).  
5. Набор языков и рынков на релиз.  
6. Требования по accessibility (цветовая слепота, крупные элементы).  
7. Нужно ли внедрять редактор уровней для внутренних нужд / UGC.

> По мере ответов на открытые вопросы документ дополнится таймлайном и приоритетами.
